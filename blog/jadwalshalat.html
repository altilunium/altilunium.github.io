Tiba-tiba, saya kepikiran untuk membuat web-spider. Kenapa? Misalkan, saya menemukan suatu blog orang penting. Konten-konten di blog tersebut banyak yang bagus. Sayangnya, blog itu tidak ada daftar isinya. Saya harus manual klik-klik satu persatu untuk mencari seluruh artikel bagus yang ada di blog itu. Melelahkan. Kalau saya punya spider sendiri, saya bisa menyuruh spider saya untuk menjelajahi seluruh link yang ada di blog itu. Saya tinggal baca laporannya saja : "daftar isi" lengkap suatu blog.

Singkat cerita, sekarang spidernya sudah jadi. Saya fork dari repositori github yang sudah ada, lalu saya ubah-ubah sedikit sesuai selera. Saat ini saya belum pede merilisnya ke publik, karena masih ada bug yang agak parah. 

Artikel ini tidak membahas mengenai web-spider yang saya buat (mungkin lain kali saja kalau bugnya sudah diperbaiki), melainkan membahas apa yang spider saya berhasil temukan.

Saya arahkan spider saya ke arah blog milik pak T. Djamaluddin (Kepala LAPAN). Saya dapat artikel ini : https://tdjamaluddin.wordpress.com/2010/12/09/program-jadwal-shalat/ . Isinya : algoritma perhitungan jadwal shalat. Berikut kutipan langsung dari artikel itu :

"Awal tahun 1990 ketika kuliah S2 dan S3 di Kyoto University, Department of Astronomy, saya menyusun program jadwal shalat berdasarkan algoritma posisi matahari. Pembuatan program itu terdorong oleh kebutuhan teman-teman mahasiswa Muslim di Jepang untuk menentukan jadwal waktu shalat di kota mereka. Internet belum banyak berkembang, selain untuk e-mail. Sehingga tidak ada sumber informasi untuk mendapatkan jadwal shalat."

Dari dulu saya penasaran bagaimana jadwal shalat itu dihitung secara astronomis. Setelah membaca artikel ini, saya penasaran untuk mencoba memprogram aplikasi jadwal shalat secara langsung.

Masalahnya, program ini ditulis dengan bahasa pemrograman yang tidak saya kenali.



Oleh sebab itu, saya memutuskan untuk menulis ulang saja semuanya di bahasa python.

Pertama-tama, masukkan koordinat tempat dan zona waktu.



Setelah itu, kita loop dari hari pertama hingga hari ke 365 dalam satu tahun.



t adalah array yang menyimpan data jadwal shalat dalam satu hari. Kita persiapkan dari awal.

Dalam loop setiap harinya, kita lakukan perhitungan jadwal shalat. Terdapat beberapa perbedaan pendapat mengenai definisi astronomis waktu shalat. Saya hanya menjelaskan definisi yang program ini gunakan. Definisi yang digunakan program ini mungkin berbeda dengan cara perhitungan lainnya.  Definisi ini diuraikan secara lengkap disini : https://tdjamaluddin.wordpress.com/2010/04/19/matahari-dan-penentuan-jadwal-shalat/ 


Subuh, mulai dari munculnya cahaya di ufuk timur menjelang terbit matahari. Secara astronomis didefinisikan sebagai astronomical dawn , kira-kira 18 derajat di bawah ufuk (108 derajat dari zenith).

Isya, ketika memudarnya cahaya merah di ufuk barat. Didefinisikan sebagai astronomical  dusk. Juga kira-kira 18 derajat di bawah ufuk.



Sekarang, waktu zuhur dan magrib. Zuhur itu tengah hari lewat sedikit. Sedangkan magrib itu matahari tenggelam lewat sedikit. Tidak boleh tepat sekali dengan tengah hari maupun matahari tenggelam, karena ada larangan shalat ketika tepat matahari terbit, matahari di tengah, dan matahari tenggelam.

Untuk zuhur, perhitungan tengah harinya menggunakan (sunrise+sunset)/2, lalu tambahkan 2/60.



Untuk magrib, tambahkan sunset dengan 2/60.



Sekarang ashar. Artikel di atas merumuskan definisi ashar sebagai "pertengahan antara zuhur dan magrib". Untuk mengetahui alasan mengapa definisi ini digunakan, silahkan baca langsung di artikelnya.



"Resep rahasia" dari program ini adalah fungsi something()



Saya tidak punya latar belakang keilmuan di astronomi. Saya hanya ada di bidang informatika. Jadi saya tidak tahu persis sebenarnya fungsi ini melakukan apa. Oleh sebab itu saya namakan fungsinya "something". Saya hanya menerjemahkan subrutin ini dari source-code aslinya ke fungsi python. Setelah saya teliti, sepertinya fungsi ini menerima masukan derajat azimuth matahari, lalu menghasilkan waktu kapan matahari ada di posisi tersebut.

Langkah terakhir, saatnya pengujian. Saya masukkan koordinat Kota Bekasi. Lalu bandingkan  dengan jadwal shalat Kota Bekasi terbitan Data Kemenag Jakarta Pusat yang diambil dari Google. Saya bandingkan data untuk bulan Maret.


Begini hasilnya (kiri : Kemenag, kanan : program ini)



Saya ambil beberapa sampelnya, untuk dibandingkan bagaimana selisihnya.



Perhitungan Kemenag mungkin saja menggunakan definisi waktu shalat dan cara perhitungan waktu yang berbeda dengan program ini. Oleh sebab itu hasilnya dapat berbeda.

Untuk subuh, program ini lebih lambat sekitar 7 menit dengan data Kemenag. Untuk sunrise, zuhur, magrib dan isya, cenderung cukup mirip. Untuk asar, bisa lebih cepat hingga 10 menit dibandingkan data Kemenag. Masalah asar ini sudah dibahas mendalam di artikel aslinya :

"Kalau pendapat ini yang digunakan, waktu salat asar akan lebih cepat sekitar 10 menit dari jadwal salat yang dibuat Departemen Agama."
